# ADR-0008 — ICP + Positioning (ขายให้ใคร · พูดว่าอะไร)

## บริบท

Phase 1–6 merged เข้า `main` หมดแล้ว (inbox รวมช่องทาง, realtime, routing, bot rule-based,
จอจัดการบอท, entitlement) และมี **ใบเสนอราคาแบบเลือกฟีเจอร์เอง** (`apps/billing` — 72 ฟีเจอร์/10 หมวด)
ที่เคาะ **ราคาแล้ว**: ค่าติดตั้ง ฿12,000–28,000 (ตามประเภทธุรกิจ) + รายเดือน ฿900–2,800 + ฟีเจอร์ที่เลือก

แต่ยังไม่เคยเคาะ 2 เรื่องที่เป็น input ของ **ทุกอย่าง** ในงาน marketing (ดู [[marketing-page-brief]] ข้อ C):
**ขายให้ใคร** (ICP) และ **ประโยคแรกจะพูดว่าอะไร** (positioning) — ถ้าไม่เคาะ จะเขียน copy แบบเอาใจทุกคน
แล้วไม่โดนใครเลย

**สัญญาณจากราคาที่เคาะไปแล้ว** (ตีความ ไม่ใช่เดา): setup หลักหมื่น = **ไม่มีใครรูดบัตรจ่ายเอง**
→ นี่คือโมเดล **sales-led / ต้องคุยก่อน** ไม่ใช่ self-serve SaaS · และค่ารายเดือน ฿900–2,800
**ต่ำกว่า SaaS per-seat ของคู่แข่งมาก** เมื่อทีมโตขึ้น → จุดต่างเชิงเศรษฐศาสตร์อยู่ตรงนี้

## การตัดสินใจ

### 1. ICP หัวหอก = **เอเจนซี่ / ผู้ดูแลหลายแบรนด์**

| ลำดับ      | กลุ่ม                                  | เหตุผล                                                                                                  |
| ---------- | -------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| **หัวหอก** | เอเจนซี่ / ธุรกิจหลายแบรนด์            | **multi-tenant คือของที่ build แล้วจริง** และคู่แข่ง SaaS คิดเงินต่อ workspace = ราคาบานสำหรับเขา       |
| secondary  | ธุรกิจบริการ / ศูนย์ซัพพอร์ต (5–20 คน) | routing + assignment ตรง pain โดยตรง · ยอมจ่ายเพราะเจ็บจริงจากสายตกหล่น                                 |
| ไว้ทีหลัง  | SME ไทย LINE-first                     | ตลาดใหญ่แต่ price sensitive สุด · setup 12k ยังแพงสำหรับหลายราย · ต้องเล่นเกม volume ซึ่งเรายังไม่พร้อม |

**ทำไมเอเจนซี่ก่อน:** deal ใหญ่สุด (base ฿28k) · 1 ดีล = ประตูสู่ลูกค้าปลายทางหลายราย (leverage) ·
คุยกับคนที่เข้าใจ tech อธิบายสั้นกว่า · และ multi-tenant ที่เรามีอยู่แล้วคือของที่เขา **ต้องจ่ายแพง**
ถ้าไปใช้เจ้าอื่น

**ข้อเสียที่ยอมรับ:** ตลาดเล็กกว่า SME · เอเจนซี่ต่อราคาเก่ง · เสี่ยงเขา build เอง
→ กันด้วยการชูว่า "ของพร้อมใช้วันนี้ ไม่ต้องจ้างทีม dev มานั่งทำ 6 เดือน"

### 2. หัวหอก positioning = **"ราคาไม่บวมตามจำนวนคน"**

ชนคู่แข่ง per-seat (SleekFlow / respond.io / Zendesk / Intercom) **ตรงๆ ด้วยตัวเลขของเราเอง** —
จ่ายค่าติดตั้งครั้งเดียว + รายเดือนคงที่ เพิ่มคนตอบแชทกี่คนราคาก็เท่าเดิม

**ทำไมอันนี้ ไม่ใช่ "รวมทุกช่องทางในจอเดียว":** อันหลังคือคำอธิบาย _category_ ที่คู่แข่งทุกเจ้าพูดเหมือนกันหมด
→ เราจะกลายเป็น "ตัวถูกกว่า" เฉยๆ · ส่วน "ราคาไม่บวมตามคน" คือข้อได้เปรียบที่ **พิสูจน์ได้ด้วยใบเสนอราคา
ที่ลูกค้ากดดูเองได้ทันที** ไม่ใช่คำโฆษณา

**ลำดับสารบนหน้าเพจ:** ราคาไม่บวมตามคน (หัวหอก) → รวมทุกช่องทางจอเดียว (บริบท/category) →
multi-tenant สำหรับเอเจนซี่ (proof ของ ICP) → ข้อมูลอยู่กับคุณ/self-host (ตัวปิดดีล ไม่ใช่ตัวเปิด)

### 3. Landing อยู่ใน `apps/billing` — **ไม่แยก app ใหม่**

`apps/billing` มี landing + builder + ใบเสนอราคาอยู่แล้ว → funnel ครบวงในที่เดียว
(เห็นเพจ → ติ๊กฟีเจอร์ → ได้ราคาทันที) และ ICP=เอเจนซี่ คือคนที่ **อยากเห็นราคาเดี๋ยวนี้**
การแยก app ที่สามมีแต่ต้นทุน maintenance โดยไม่ได้อะไรเพิ่ม

> ⚠️ `apps/billing` อยู่ **นอก** pnpm workspace/`pnpm gate` (ตั้งใจ — เป็นเครื่องมือ go-to-market
> ไม่ใช่ product) → verify ด้วย `npm run lint` + `npm run build` ของตัวเอง

### 4. กฎเนื้อหา — **ห้ามขายเกินจริง** (บังคับ ไม่ใช่คำแนะนำ)

- ❌ **ห้าม testimonial/รีวิว/โลโก้ลูกค้าที่ไม่มีจริง** — ของเดิมมี "คุณสมชาย/คุณวิมล/คุณพิชัย" 5 ดาว
  เป็นของแต่งล้วน → **ถอดออก** แทนด้วย use-case card ที่พูดในนามเราเอง ("ระบบนี้เหมาะกับ…")
- ❌ **ห้ามเคลมช่องทางที่ยังไม่ได้ต่อว่า "มีแล้ว"** — ต่อจริง = **เว็บ widget + LINE** เท่านั้น ·
  ที่เหลือ (Messenger / IG / WhatsApp / Telegram / อีเมล) พูดได้แค่ **"สั่งเสียบเพิ่มได้"**
- 🟡 **AI reply = "beta"** จนกว่าจะยิง Anthropic จริงสักครั้ง · bot rule-based = เคลมได้เต็มปาก

## ผลที่ตามมา

- copy ทุกหน้าใน `apps/billing` ต้องเรียงสารตามข้อ 2 และผ่านกฎข้อ 4
- ประเภทธุรกิจ `agency` ใน [mockFeatures.ts](../../../apps/billing/src/data/mock/mockFeatures.ts)
  กลายเป็น **default/แนะนำ** ไม่ใช่แค่ตัวเลือกสุดท้ายในลิสต์
- ต้องมีตัวเลขเทียบ per-seat ให้เห็นภาพ (ยังไม่ได้ทำ — ต้องหาราคาคู่แข่งจริงมาอ้างอิงก่อน ห้ามมั่วเลข)
- ยังค้าง: screenshot/เดโม inbox จริง (ใช้ [[inbox-e2e-harness]] ถ่ายได้), lead funnel ไปไหน

## ทางเลือกที่พิจารณาแล้วไม่เอา

| ทางเลือก                   | ทำไมไม่เอา                                                                           |
| -------------------------- | ------------------------------------------------------------------------------------ |
| ICP = SME LINE-first       | price sensitive สุด ต้องขายทีละรายจำนวนมาก — เกินกำลัง solo+AI ตอนนี้                |
| positioning = self-host    | คนที่แคร์เรื่องนี้จริงมีน้อย และมักเป็นองค์กรใหญ่ที่ขายรอบยาว → เก็บไว้เป็นตัวปิดดีล |
| positioning = custom build | ขายเป็น "บริการ" ไม่ใช่ "ผลิตภัณฑ์" → รายได้ผูกกับเวลาคน scale ไม่ได้                |
| แยก app marketing ใหม่     | ต้นทุน maintenance เพิ่ม โดย funnel ขาดตอนจาก builder ที่มีอยู่แล้ว                  |
